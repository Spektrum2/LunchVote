# LunchVote

LunchVote — это REST API для голосования за то, где сегодня пообедать. Проект разработан с использованием **Spring Boot**, **Spring MVC**, **Hibernate** и **Spring Data JPA**, что позволяет управлять ресторанами, их ежедневными меню и голосованием пользователей. Данный проект не содержит фронтенд, поэтому API предназначено для интеграции с отдельно разрабатываемым клиентским приложением.

## Особенности

- **Роли пользователей:** два типа пользователей — администраторы и обычные.
- **Управление ресторанами:** администраторы могут создавать, редактировать и удалять рестораны.
- **Управление меню:** для каждого ресторана администратор задаёт меню на текущий день (обычно 2–5 позиций, каждая из которых содержит наименование блюда и цену). Меню обновляется ежедневно.
- **Система голосования:** обычные пользователи могут голосовать за понравившийся ресторан для обеда сегодня. Один пользователь может отдать только один голос в день.
- **Изменение голоса:** если пользователь голосует повторно в один и тот же день, то до **11:00** считается, что он изменил своё решение; после **11:00** голос изменить нельзя.
- **Гибкость:** API предназначено для использования фронтенд-разработчиком при создании клиентского приложения.

## Используемые технологии

- **Spring Boot** – быстрая разработка и автоматическая конфигурация.
- **Spring MVC** – реализация RESTful веб-сервисов.
- **Spring Data JPA / Hibernate** – работа с базой данных через ORM.
- **Spring Security** – аутентификация и авторизация.
- **Swagger/OpenAPI 3** – интерактивная документация API.
- **H2** (или другая СУБД) – хранение данных.

## Требования

- **Java 11+**
- **Maven 3.6+** (или другая система сборки)
- СУБД – по умолчанию используется **H2** для разработки
